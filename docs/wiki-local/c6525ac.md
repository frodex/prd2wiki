## `mem_` ID Format — Zero-Padded Base-36 (R8-1, R9-2)

```go
func generateMemoryID() string {
    for {
        ts := strconv.FormatInt(time.Now().UnixNano(), 36)
        // Zero-pad to 14 chars for lexicographic monotonicity
        for len(ts) < 14 {
            ts = "0" + ts
        }
        id := "mem_" + ts + randomSuffix(4)
        if _, exists := t.records[id]; !exists {
            return id
        }
    }
}
```

**Format:** `mem_` + 14-char zero-padded base-36 nanosecond timestamp + 4-char random hex.

**Why zero-pad:** Unpadded `strconv.FormatInt(n, 36)` is NOT lexicographically monotonic at digit-width boundaries:

- `36^11 - 1` → `"zzzzzzzzzzz"` (11 z's, 11 chars)
- `36^11` → `"100000000000"` (1 followed by 11 zeros, **12 chars**)
- String compare: `"zzzzzzzzzzz" > "100000000000"` (z > 1 at first char) — **wrong order** even though 36^11 > 36^11 - 1

Zero-padding to a fixed width fixes this:
- `"000zzzzzzzzzzz"` (3 zeros + 11 z's = 14 chars)
- `"00100000000000"` (2 zeros + 12 chars = 14 chars)
- String compare: `"000z..." < "001..."` — **correct**

**Why 14 chars:** Current (2026) nanosecond timestamps (~1.775 × 10^18) produce **12 base-36 digits** unpadded. Max int64 (9.22 × 10^18) produces **13 base-36 digits**. Padding to 14 covers all possible int64 nanosecond values with room to spare.

**Sub-nanosecond tie-break (R9-3):** Two IDs generated in the same nanosecond differ only in the 4-char random suffix. Lexicographic ordering of the suffix is **deterministic but not temporal** — it's an arbitrary tie-break, which is acceptable for dual-current repair (both rows are "equally recent").

**Hard contract:** The dual-current repair tie-break depends on zero-padded lex monotonicity. If the ID scheme changes, the tie-break must be reviewed.

## Write Strategy: New Chain Forward

Add new head first (always safe). Demote old head second (Lance upsert on OLD row). New head never at risk.

```
Step 1 — Add new head:     ...← mem_r8(v4, current)     mem_x1(v5, current)
Step 2 — Demote old head:  ...← mem_r8(v4, superseded) ← mem_x1(v5, current)
Step 3 — Update index:     page_uuid_index → mem_x1, .link line 2 → mem_x1
```

### Crash recovery

| Crash point | Recovery |
|-------------|----------|
| After step 1, before step 2 | Two "current" → highest version wins, tie-break by zero-padded ID |
| After step 2 Delete, before step 2 Add | New head correct, old head missing → chain gap, acceptable |
| After step 2 | No repair needed |

### Code

```go
func (s *MemoryStore) versionAndStore(ctx context.Context, pageUUID, newContent string, opts ...StoreOption) (string, error) {
    oldHead := s.table.GetCurrentByPageUUID(pageUUID)
    if oldHead == nil {
        return s.createFirst(ctx, pageUUID, newContent, opts...)
    }

    newVec, err := s.embedder.Embed(ctx, newContent)
    if err != nil { return "", err }

    newHead := &MemoryRecord{
        ID: generateMemoryID(), PageUUID: pageUUID,
        Version: oldHead.Version + 1, VersionStatus: "current",
        Supersedes: oldHead.ID, Content: newContent, Vector: newVec,
        Namespace: oldHead.Namespace,
    }
    applyOptions(newHead, opts)

    demoted := oldHead.Copy()
    demoted.VersionStatus = "superseded"
    demoted.SupersededBy = newHead.ID

    if err := s.lance.Add(ctx, newHead); err != nil { return "", err }
    if err := s.lance.Upsert(ctx, demoted); err != nil {
        slog.Warn("demotion failed — startup will fix", "new", newHead.ID, "old", oldHead.ID)
    }

    s.table.Put(newHead)
    s.table.Put(demoted)
    return newHead.ID, nil
}
```

### Startup dual-current repair

```go
func (s *MemoryStore) repairDualCurrents() {
    byPage := map[string][]*MemoryRecord{}
    for _, rec := range s.allRecords() {
        if rec.PageUUID != "" && rec.VersionStatus == "current" {
            byPage[rec.PageUUID] = append(byPage[rec.PageUUID], rec)
        }
    }
    for pageUUID, currents := range byPage {
        if len(currents) <= 1 { continue }
        sort.Slice(currents, func(i, j int) bool {
            if currents[i].Version != currents[j].Version {
                return currents[i].Version > currents[j].Version
            }
            return currents[i].ID > currents[j].ID // zero-padded = time-ordered
        })
        winner := currents[0]
        for _, loser := range currents[1:] {
            loser.VersionStatus = "superseded"
            loser.SupersededBy = winner.ID
            s.lance.Upsert(ctx, loser)
            slog.Info("repaired dual-current", "page", pageUUID, "kept", winner.ID, "demoted", loser.ID)
        }
    }
}
```

## Implementation Order

1. `generateMemoryID()` — zero-padded base-36 (14 chars) + 4-char random suffix
2. Single LanceDB table — pre-flight item 6
3. Secondary index: `page_uuid → current record ID`
4. Version fields + `version_status`
5. `Store()` returns `(string, error)`. MCP adds `record_id`.
6. New-chain-forward versioning
7. Startup dual-current repair (version + zero-padded ID tie-break)
8. Delete: promote, ghost, dissolve, cascade, -chain, -history
9. Search: regular + history_count, deep + chain
10. MCP handler for memory_delete
11. Integration tests (including ID boundary-value and same-nanosecond cases)

## Invariants

- **One current per page_uuid** — single-writer mutex + startup repair
- **page_uuid = ""** for entities
- **page_uuid is stable** — `mem_` ID changes each edit
- **New head added BEFORE old head demoted**
- **No Delete+Add on new head** — demotion upsert on old head only
- **Crash = at worst two "current" rows** — highest version wins, zero-padded ID tie-break
- **`mem_` IDs zero-padded to 14 base-36 chars** — lex order = time order across digit boundaries
- **Sub-nanosecond ordering is suffix-based** — deterministic but arbitrary, acceptable for repair
- **Embed before any Lance write**
- **Single-writer** through Table mutex
- **Git is source of truth**
- **IDs never reused**
- **Table: `pippi_memory`**, **field: `version_status`**
