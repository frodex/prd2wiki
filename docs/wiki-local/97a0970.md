## Dot-Files in the Tree

| File | Where | Format |
|------|-------|--------|
| `.uuid` | Project root | Line 1: UUID, Line 2: display name |
| `{name}.link` | Any dir | Line 1: page UUID, Line 2: current `mem_` ID (updated each edit), Line 3: title |
| `.access` | Any dir | `read:` / `write:` rules (proposed) |
| `.301` | Old location | Target path (permanent redirect) |
| `.302` | Old location | Target path (temporary redirect) |

**`.link` line 2 is volatile.** Under new-chain-forward versioning, the head `mem_` ID changes on every edit. The wiki updates `.link` line 2 after each successful `syncToLibrarian`. `page_uuid` (line 1) is the stable identifier.

## Rules for Agents

1. **Source code** in `/srv/`. **Never edit** `/opt/`.
2. **Build in source, install to opt.**
3. **`.go.bak` = dead code.** Ignore.
4. **`LD_LIBRARY_PATH`** for LanceDB tests.
5. **SIGTERM** to stop pippi-librarian. Not `kill -9`.
6. **Service keys** issued before server starts.
7. **Tree is source of truth.** Projects from `.uuid`, pages from `.link`. Not config.
8. **`page_uuid` never changes.** `mem_` head ID changes every edit. Use `page_uuid` for lookups.
9. **One LanceDB table** (`pippi_memory`). All `mem_` IDs. `record_type` column.
10. **Remote librarian client** at `internal/libclient/`, NOT `internal/librarian/`.
