# REFERENCE: Librarian Tools — What Agents Get Back

**Date:** 2026-04-11 (R11: restore memory_delete detail)
**Source:** Design docs, not live results (implementation pending)
**Related:** [Version-Aware Memory](/projects/default/pages/c6525ac) (c6525ac), [Unified Identity](/projects/default/pages/de836ff) (de836ff), [Master Plan](/projects/default/pages/8634f02) (8634f02)

## What Exists Today

Agents have one search tool returning raw content blobs with file-path keys. No title, snippet, or version info.

---

## Proposed: Four Tools, UUID-Based

All addressing uses `page_uuid` (`.link` line 1) as the stable identifier. `mem_` IDs change every edit.

### Namespace

`wiki:{project-uuid}` — isolates projects in search.

---

### memory_search — find pages

**Regular:** Returns current versions with `page_uuid`, `record_id` (volatile), title, snippet, score, `history_count`.

**Deep (`deep: true`):** Returns `page_uuid` groups with per-version scores. Ghosts skipped.

`record_id` in results is the current head at query time — may change if page is edited. Use `page_uuid` for follow-up.

---

### memory_get — read a specific page or record

**By page UUID (current — recommended):**
```json
{"name": "memory_get", "args": {"page_uuid": "550e8400-..."}}
```

**By record ID (specific version):**
```json
{"name": "memory_get", "args": {"id": "mem_0hf5e3a8k9m2p1a3b7"}}
```

**With history:**
```json
{"name": "memory_get", "args": {"page_uuid": "550e8400-...", "include_history": true}}
```

**Precedence:** `page_uuid` and `id` are **mutually exclusive**. If both provided, return error.

Each version in history has a unique `record_id`.

---

### memory_store — create or update a page

Each store creates a **new** head row (new `mem_` ID). Old head demoted to "superseded."

```json
{"name": "memory_store", "args": {"namespace": "wiki:660f9500-...", "page_uuid": "550e8400-...", "content": "...", "metadata": {...}}}
```

**Response:**
```json
{"record_id": "mem_0hf5e3a9l0n3q2d8e4", "version": 5, "created": false}
```

---

### memory_delete — remove records

```json
{"name": "memory_delete", "args": {"id": "mem_0hf5e3a8k9m2p1a3b7"}}
{"name": "memory_delete", "args": {"id": "mem_0hf5e3a8k9m2p1a3b7", "chain": true}}
{"name": "memory_delete", "args": {"id": "mem_0hf5e3a8k9m2p1a3b7", "history": true}}
```

**Response:**
```json
{"affected": 3, "promoted": "mem_0hf5e3a7j8l1o0x2c6"}
```

**Context-aware behavior** (full semantics in [c6525ac § Delete](/projects/default/pages/c6525ac)):

| What the ID points to | What happens |
|----------------------|-------------|
| **Standalone** (no chain) | Gone from database |
| **Current** (has chain) | Promote previous version to current. `.link` line 2 + index updated to promoted ID. |
| **Interior** version | Ghost it — clear content + vector, keep chain links |
| **Tail** (oldest) | Dissolve + cascade upstream ghosts (stop at first non-ghost) |

**Flags:**
- **`-chain`**: Delete target + everything older toward the tail. Everything newer stays.
- **`-history`**: Keep current, delete all history. Current becomes standalone.

**Response fields:**
- `affected`: number of records removed or ghosted
- `promoted`: record ID promoted to current (only when deleting a current with chain). Empty if standalone or flag-based.

## Key design decisions

| Decision | Rationale |
|----------|-----------|
| Extension fields surfaced inline | Agents see `source_commit`, not `ext.source_commit`. |
| Namespace is `wiki:{project-uuid}` | Isolates projects. Uses UUID, not name. |
| **`page_uuid` for lookups** | Stable. `mem_` ID is volatile. Index resolves transparently. |
| **`page_uuid` / `id` mutually exclusive** in `memory_get` | "Get current" vs "get specific version." |
| **Head `mem_` ID changes every edit** | New-chain-forward. `.link` line 2 updated each sync. |
| `record_id` in every response | Track specific versions, but use `page_uuid` for "get current." |
| Sub-nanosecond tie-break is arbitrary | Same-ns IDs differ by random suffix. |
| Delete is context-aware | Position in chain determines behavior. Explicit flags for bulk operations. |

## Four tools

| Tool | Default | With flag | Returns |
|------|---------|-----------|---------| 
| `memory_search` | Current only | `deep: true` → history | `page_uuid`, snippet, score, history_count |
| `memory_get` | Current (by page_uuid) or specific (by id) | `include_history` | Full content + version chain |
| `memory_store` | New head per edit | metadata | **New** `record_id` + version |
| `memory_delete` | Context-aware (promote/ghost/dissolve) | `chain` / `history` | Affected count + promoted ID |
